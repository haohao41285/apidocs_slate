    METHOD OF OPERATION: Vietnamese network operators send Ads messages according to a set of at least 100 phone numbers / 1 message content / 1 time of sending. Therefore, if the customer's system can send QC message by the method of calling API with multiple phone numbers 1 time (at least 100 numbers), you can connect to VGs' system according to instructions.

    #HTTP Request

    POST https://qc.vietguys.biz/api/sendsms.php

    #Query Parameters

    Parameter Default Description
    phone false The phone number receiving the message, many phone numbers separated by commas.
    from true Sender's name. Ex: VIETGUYS, 6x89,...
    sms true Sms's content
    u false Account
    pwd false Program Code
    day true Date to send sms, Format: yyyy-mm-dd (If empty, sms be sent as soon as possible)
    time false Time to send sms, Format: hh:mm

    const kittn = require('kittn');

    The above command returns CODE or JSON like this:

      00: Successfully
  01: Server Vietguys's busy
  02: The input information has not been fully entered.
  03: Account does not exist
  04: Phone number is not correct
  05: Account is locked.
  06: Authentication information is not correct
  07: Querying from undeclared IP address
  08: Invalid topup denomination value.
  09: Duplicate transaction code id (do not check default, must report IT settings)
  10: Account topup credit end.

    #HTTP Request

    With SSL(recommended): GET/POST https://cloudsms2.vietguys.biz:4438/api/topup/index.php
    Without SSL: GET/POST https://sms.vietguys.biz/api/topup/index.php

    #Query Parameters

    Parameter Default Description
    u false SMS account username
    pwd false SMS account passcode (not password for web login)
    phone false Recipient’s phone number must be in international format (e.g. 84901234567)
    amount false Amount topup: 10000, 20000, 50000, 100000, 200000, 500000
    tid false Transaction id of partner, used for cross-check (Required).
    type false Type of topup to load, type = data to load capacity. (Leave blank by default is deposit)

    #TopUp Status

    Value Meaning
    2 Sent to the network operator
    1 Successfully TopUp
    5 Card Successfully
    4 Failed
    3a Retry topup to the network operator
    3b Retry card to the network operator

    #HTTP Request

    With SSL(recommended): GET/POST https://cloudsms.vietguys.biz:4438/api/index_ott.php
    Without SSL: GET/POST http://cloudsms.vietguys.biz:8088/api/index_ott.php

    #Query Parameters

    Parameter Default Description
    u false SMS account username
    pwd false SMS account passcode (not password for web login)
    phone false Recipient’s phone number must be in international format (e.g. 84901234567)
    bid false Message ID from MO
    viber false Viber content
    from_viber false Viber Brand
    json false Return type, set to 1 will return a json string
    bid false Message ID from MO.
    actiontext false Text in Call-to-action button
    actionlink true Path when click Call-to-action button
    viberimg true Image shown below content

    (*) Note with 03 parameters actiontext, actionlink, viberimg if using 03 simultaneous use or the system will return an error. 02 parameters actionlink, viberimg can run the FILTER_URL test function to return an error (if the path structure is wrong)
